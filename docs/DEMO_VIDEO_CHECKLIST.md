# Demonstration Video Checklist

Use this flow to record the required working prototype demonstration.

## 1) Setup and startup

- Show repository clone and folder structure
- Show backend `.env` setup from `backend/.env.example`
- Show frontend `.env` setup from `frontend/.env.example` (optional)
- Run backend and frontend servers
- Open app in browser

## 2) Authentication and roles

- Register a new normal user
- Login as normal user
- Login as admin (`admin@recipehub.com`)
- Show role-restricted behavior (admin dashboard access control)

## 3) Core recipe lifecycle (CRUD)

- Create a new recipe
- View recipe details
- Edit own recipe
- Delete own recipe
- Rate a recipe (1-5 stars)

## 4) Meal planner flow

- Add recipe to a day/meal slot
- Update slot by adding another recipe
- Remove one slot
- Clear full planner

## 5) Bookmarks and profile

- Bookmark/unbookmark a recipe
- Open bookmarks page
- Update profile fields (name/bio/avatar)

## 6) Admin flows

- Open admin dashboard stats
- View all users
- Delete one user
- View all recipes (admin listing)

## 7) API documentation and integration proof

- Open Swagger UI (`/api/docs`)
- Execute one public endpoint and one protected endpoint via Swagger/Postman
- Show frontend action and matching backend response

## 8) Closing

- Summarize architecture briefly
- Mention AI assistance areas and learning outcomes
- Share Google Drive video link in final submission
